## What I'm Building

- **Multi-AZ VPC architectures** with Terraform — public/private subnets, NAT Gateway, route tables, least-privilege security groups
- **Auto Scaling Groups** with CPU-based target tracking policies and CloudWatch monitoring
- **CI/CD pipeline for IaC** using GitHub Actions: `fmt` → `validate` → `tfsec` → `plan`
- **Operational documentation**: runbooks, Architecture Decision Records (ADRs), cost governance tables

## Featured Project

### [AWS Terraform Portfolio](https://github.com/Victor-David-Medina/aws-terraform-portfolio)

Production-style AWS VPC stack built as a Cloud Operations portfolio project. Every component reflects real-world enterprise infrastructure patterns.

| Component | Details |
|---|---|
| Networking | Multi-AZ VPC with public/private subnet segmentation across 2 AZs |
| Compute | Auto Scaling Group (2–6 instances) with CPU-based target tracking |
| Egress | NAT Gateway for controlled outbound access from private subnets |
| Security | GuardDuty threat detection, least-privilege IAM and security groups |
| Observability | CloudWatch alarms for scaling visibility and cost optimization |
| Cost Controls | AWS Budget alerts for cost governance |
| CI/CD | GitHub Actions: `terraform fmt` → `validate` → `tfsec` → `plan` |
| Day-2 Ops | Runbook + ADRs + cost table included in the repo |
